## 本地项目规则 / Local Project Rules

### 规则1: 静默期禁止事项

**🚫 本周内绝对禁止:**

1. ❌ 联系Kimi官方 (邮件、社交媒体、GitHub issue)
2. ❌ 公开发布到社区 (V2EX、掘金、知乎等)
3. ❌ 在竞对仓库活跃 (避免引起注意)
4. ❌ 透露项目精确数据 (skills数量、运营策略)
5. ❌ 使用项目身份对外发言

**✅ 本周内专注:**

1. ✅ 本地skills扩充 (数量目标200)
2. ✅ 完善内部文档 (质量认证体系)
3. ✅ 优化工具链 (质量检查、自动化)
4. ✅ 准备发布材料 (一周后使用)
5. ✅ 本地测试验证 (确保质量)

### 规则2: Skills开发规范

**批量创建skills原则:**

```
优先级1: 移植竞对热门skills (本地化改造)
优先级2: 填充细分赛道 (每个细分至少3个)
优先级3: 完善现有skills (升级至AA级)
```

**质量底线:**
- 所有新skills必须通过quality-check.py检查
- 最低要求达到B级 (≥40分)
- 推荐达到A级 (≥60分)
- 核心skills必须达到AA级 (≥80分)

### 规则3: 文档规范

**所有文档必须:**

1. 双语编写 (中文在前，英文在后)
2. 模板化 (使用templates/skill-template)
3. 完整性 (安装/使用/示例/参数/FAQ)
4. 一致性 (术语统一，格式规范)

### 规则4: 提交规范

**静默期提交信息格式:**

```
类型: 中文描述 / English description

详细中文说明
Detailed English description

静默期 / Stealth: Week 1 Day X
```

### 规则5: 数据保密

**禁止外泄数据:**
- 当前skills精确数量
- 质量检查详细评分
- 运营策略细节
- 商业计划内容

**对外统一口径:**
> "我们正在建设全球最大的中文Kimi Skills仓库，具体数据敬请期待。"
