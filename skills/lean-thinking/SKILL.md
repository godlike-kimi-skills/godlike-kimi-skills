# Lean Thinking

**起源**: Toyota Production System (1950s)  
**核心著作**: 《精益思想》(Womack & Jones, 1996)  
**类型**: 执行与迭代  
**优先级**: P0

---

## 核心概念

### 精益五原则

```
┌─────────────────────────────────────────────┐
│             精益思想五原则                   │
├─────────────────────────────────────────────┤
│                                             │
│  1. 价值 (Value)                            │
│     ├─ 从客户视角定义价值                    │
│     └─ 客户愿意付费的才算价值                │
│                                             │
│  2. 价值流 (Value Stream)                    │
│     ├─ 识别创造价值的所有活动                │
│     └─ 区分增值/非增值/浪费                  │
│                                             │
│  3. 流动 (Flow)                             │
│     ├─ 价值活动连续流动                      │
│     └─ 消除中断和等待                        │
│                                             │
│  4. 拉动 (Pull)                             │
│     ├─ 由客户需求触发                        │
│     └─ 而非预测推动                          │
│                                             │
│  5. 尽善尽美 (Perfection)                    │
│     ├─ 持续追求完美                          │
│     └─ 通过不断改善逼近                      │
│                                             │
└─────────────────────────────────────────────┘
```

## 浪费识别 (Muda)

### 七大浪费

```
TIMWOOD 记忆法:

T - Transportation (运输浪费)
    ├─ 不必要的产品/信息搬运
    ├─ 远距离协作
    └─ 频繁的系统切换
    
I - Inventory (库存浪费)
    ├─ 过量的原材料/在制品
    ├─ 未完成的工作堆积
    └─ 信息冗余存储
    
M - Motion (动作浪费)
    ├─ 不必要的人员移动
    ├─ 寻找工具/信息
    └─ 复杂的操作流程
    
W - Waiting (等待浪费)
    ├─ 等待审批/反馈
    ├─ 等待他人完成
    └─ 系统响应等待
    
O - Overproduction (过度生产)
    ├─ 生产超出需求
    ├─ 过早开发功能
    └─ 信息的过度加工
    
O - Overprocessing (过度加工)
    ├─ 超出需求的精度
    ├─ 不必要的审批环节
    └─ 重复的质量检查
    
D - Defects (缺陷浪费)
    ├─ 返工/修复
    ├─ 错误导致的损失
    └─ 客户投诉处理

+ 第八大浪费:
N - Non-utilized Talent (人才浪费)
    ├─ 员工能力未充分发挥
    ├─ 创意建议被忽视
    └─ 缺乏授权和成长
```

### 知识工作浪费

```
软件/服务业的特殊浪费:

1. 部分完成的工作
   ├─ 未完成的特性
   ├─ 未集成的代码
   ├─ 未部署的变更
   └─ 未验证的假设

2. 额外流程
   ├─ 不必要的审批
   ├─ 冗余的文档
   ├─ 过度的汇报
   └─ 形式主义的会议

3. 额外功能
   ├─ 用户不需要的功能
   ├─ 过度工程化
   ├─ 镀金需求
   └─ 过早优化

4. 任务切换
   ├─ 多任务并行
   ├─ 频繁打断
   ├─ 上下文切换成本
   └─ 注意力碎片化

5. 等待
   ├─ 等待决策
   ├─ 等待资源
   ├─ 等待信息
   └─ 等待部署窗口

6. 移动
   ├─ 团队分散
   ├─ 系统切换
   ├─ 信息查找
   └─ 交接传递

7. 缺陷
   ├─ Bug修复
   ├─ 技术债务
   ├─ 理解偏差
   └─ 客户不满意
```

## 价值流映射

### 价值流分析

```
当前状态图:

客户需求 → [营销] → [销售] → [开发] → [测试] → [部署] → [运维]
              ↓        ↓        ↓        ↓        ↓        ↓
            3天      5天      14天     7天      2天      持续
              ↓        ↓        ↓        ↓        ↓        ↓
           增值     增值     增值     增值     增值     增值
           20%      30%      40%      30%      20%      80%
              ↓        ↓        ↓        ↓        ↓        ↓
           等待     等待     等待     等待     等待     
           80%      70%      60%      70%      80%      

总周期时间: 31天
增值时间: 约6天 (19%)
浪费: 约25天 (81%)

未来状态图目标:
总周期时间: 7天
增值比例: >50%
```

### 价值流优化

```
优化步骤:
1. 识别价值: 客户真正要什么?
2. 绘制当前: 记录实际流程和时间
3. 识别浪费: 标记非增值活动
4. 设计未来: 消除浪费后的理想流程
5. 实施改进: 分步实现未来状态
6. 持续完善: 重复以上步骤
```

## 精益工具箱

### 1. 5S 工作场所组织

```
1. 整理 (Seiri - Sort)
   ├─ 区分需要和不需要的
   ├─ 丢弃/归档不需要的
   └─ 目标: 只保留必要物品

2. 整顿 (Seiton - Set in Order)
   ├─ 物品各归其位
   ├─ 标识清晰
   └─ 目标: 30秒内找到任何物品

3. 清扫 (Seiso - Shine)
   ├─ 清洁工作区
   ├─ 检查设备状态
   └─ 目标: 异常立即显现

4. 清洁 (Seiketsu - Standardize)
   ├─ 将前3S标准化
   ├─ 制定规范和检查表
   └─ 目标: 维持前3S成果

5. 素养 (Shitsuke - Sustain)
   ├─ 养成习惯
   ├─ 持续改进
   └─ 目标: 成为组织文化

数字工作区5S:
├─ 文件命名规范
├─ 文件夹结构清晰
├─ 桌面整洁
├─ 邮件及时归档
└─ 知识库维护
```

### 2. 看板 (Kanban)

```
看板三原则:
1. 可视化工作流
   ├─ 待办 → 进行中 → 完成
   ├─ 限制进行中的工作数 (WIP)
   └─ 每个人都可见

2. 限制WIP
   ├─ 每人同时最多3个任务
   ├─ 团队同时最多X个任务
   └─ 完成一个才能开始下一个

3. 管理流动
   ├─ 监控周期时间
   ├─ 识别瓶颈
   └─ 持续优化流程

看板示例:
┌─────────┬─────────┬─────────┬─────────┐
│  Backlog│  To Do  │   Doing │   Done  │
│         │   [3]   │   [2]   │         │
├─────────┼─────────┼─────────┼─────────┤
│ 任务A   │ 任务C   │ 任务E   │ 任务G   │
│ 任务B   │ 任务D   │ 任务F   │ 任务H   │
│ 任务... │ 任务... │         │ 任务... │
└─────────┴─────────┴─────────┴─────────┘
        [WIP限制: 3]   [WIP限制: 2]
```

### 3. 快速换型 (SMED)

```
Single-Minute Exchange of Die
目标: 换型时间 < 10分钟

内部作业 (必须停机):
├─ 代码部署
├─ 数据库迁移
└─ 配置文件更新

外部作业 (可提前准备):
├─ 代码审查
├─ 测试用例准备
├─ 回滚方案准备
└─ 通知相关人员

优化步骤:
1. 区分内部/外部作业
2. 将内部转化为外部
3. 简化内部作业
4. 简化外部作业
5. 持续改进

软件应用:
├─ 特性开关 (Feature Toggle)
├─ 蓝绿部署
├─ 金丝雀发布
└─ 自动化测试
```

### 4. 持续改善 (Kaizen)

```
Kaizen活动:
1. 选择主题
2. 现状分析
3. 原因探究
4. 制定对策
5. 实施改进
6. 效果确认
7. 标准化
8. 下一步计划

改善提案制度:
├─ 鼓励每个人提建议
├─ 快速评估和实施
├─ 认可和奖励贡献
└─ 积累小改进为大变革

PDCA + Kaizen = 持续改善循环
```

## 精益转型路径

```
阶段1: 基础建立 (3-6个月)
├─ 培训: 全员精益思想培训
├─ 试点: 选择一个团队/流程试点
├─ 可视化: 建立看板系统
└─ 度量: 建立基础指标

阶段2: 流程优化 (6-12个月)
├─ 价值流: 绘制并优化价值流
├─ 浪费消除: 系统消除七大浪费
├─ 拉动系统: 建立按需生产
└─ 质量内建: 源头质量控制

阶段3: 文化深化 (12-24个月)
├─ 自働化: 智能自动化
├─ 持续改善: Kaizen成为习惯
├─ 全员参与: 每个人都是改善者
└─ 供应链: 扩展至合作伙伴

阶段4: 尽善尽美 (持续)
├─ 零缺陷目标
├─ 零库存目标
├─ 零浪费目标
└─ 持续逼近理想状态
```
