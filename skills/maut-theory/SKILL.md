# Multi-Attribute Utility Theory (MAUT)

**理论基础**: Keeney & Raiffa (1976)  
**类型**: 多目标决策  
**优先级**: P1

---

## 核心理论

### 为什么需要MAUT

传统决策的问题:
- 多目标不可比 (利润 vs 风险 vs 社会影响)
- 主观判断缺乏一致性
- 群体决策难以达成共识

MAUT的解决:
> 将不可比的多维目标转化为统一的效用值，实现理性排序

### 理论基础

**多属性效用函数**:
```
U(X) = U(X₁, X₂, ..., Xₙ)

其中:
- X = 方案
- Xᵢ = 第i个属性的表现
- U(X) = 方案的总效用值
```

**可加性效用函数** (最常见):
```
U(X) = Σ wᵢ × uᵢ(Xᵢ)

其中:
- wᵢ = 第i个属性的权重 (Σwᵢ = 1)
- uᵢ = 第i个属性的效用函数 (0-1标准化)
```

## MAUT实施步骤

### 步骤1: 目标层次结构化

```
决策目标: 选择最优项目

├─ 一级维度: 财务收益 (w=0.4)
│  ├─ 指标1: NPV (w₁=0.6)
│  ├─ 指标2: IRR (w₂=0.3)
│  └─ 指标3: 回收期 (w₃=0.1)
│
├─ 一级维度: 风险评估 (w=0.3)
│  ├─ 指标4: 市场风险 (w₄=0.4)
│  ├─ 指标5: 技术风险 (w₅=0.4)
│  └─ 指标6: 政策风险 (w₆=0.2)
│
├─ 一级维度: 战略契合 (w=0.2)
│  ├─ 指标7: 核心竞争力 (w₇=0.5)
│  └─ 指标8: 市场定位 (w₈=0.5)
│
└─ 一级维度: 实施难度 (w=0.1)
   ├─ 指标9: 技术难度 (w₉=0.6)
   └─ 指标10: 资源需求 (w₁₀=0.4)
```

### 步骤2: 确定权重

**权重确定方法**:

1. **直接分配法**: 专家直接给出权重
2. **两两比较法**: AHP层次分析法
3. ** swing权重法**: 从最差到最好的改进价值

```
Swing权重法示例:

基准: 所有指标取最差值
问题: 哪个指标提升到最好值最有价值?

排序结果:
1. NPV (100分)
2. 市场风险 (80分)
3. 核心竞争力 (70分)
4. ...

权重计算:
- NPV: 100/400 = 0.25
- 市场风险: 80/400 = 0.20
- 核心竞争力: 70/400 = 0.175
- ...
```

### 步骤3: 构建单属性效用函数

**效用函数类型**:

```
1. 线性函数 (风险中性):
   u(x) = (x - x_min) / (x_max - x_min)

2. 指数函数 (风险厌恶):
   u(x) = (1 - e^(-rx)) / (1 - e^(-r))

3. 对数函数 (强风险厌恶):
   u(x) = ln(x/x_min) / ln(x_max/x_min)
```

**示例: NPV效用函数**

```
NPV范围: -100万 ~ +500万

效用函数 (风险厌恶):
├─ NPV = -100万: u = 0
├─ NPV = 0: u = 0.3 (亏损厌恶)
├─ NPV = 100万: u = 0.6
├─ NPV = 300万: u = 0.85
└─ NPV = 500万: u = 1.0
```

### 步骤4: 计算总效用

```
方案A效用计算:

| 指标 | 表现 | 单效用u | 权重w | w×u |
|------|------|---------|-------|-----|
| NPV | 200万 | 0.7 | 0.15 | 0.105 |
| IRR | 25% | 0.8 | 0.075 | 0.060 |
| 市场风险 | 低 | 0.9 | 0.12 | 0.108 |
| 技术风险 | 中 | 0.6 | 0.12 | 0.072 |
| 核心竞争力 | 高 | 0.85 | 0.10 | 0.085 |
| 技术难度 | 低 | 0.9 | 0.06 | 0.054 |
| ... | ... | ... | ... | ... |

总效用 U(A) = Σ(wᵢ×uᵢ) = 0.72
```

### 步骤5: 一致性检验

```
检验项目:

1. 权重和检验: Σwᵢ = 1.0 ✓

2. 敏感性分析:
   - 权重±20%变化时，排序是否稳定?
   - 若不稳定，需重新审视权重

3. 极端值检验:
   - 检查各方案在各指标上的表现
   - 识别是否有"一票否决"项

4. 逻辑一致性:
   - 直觉判断与计算结果是否一致?
   - 不一致时检查模型假设
```

## Kbot应用场景

### 场景1: 技术方案选型

```
决策: 选择前端框架

候选: React vs Vue vs Angular

维度与权重:
├─ 开发效率 (0.25)
├─ 生态系统 (0.20)
├─ 性能表现 (0.20)
├─ 学习成本 (0.15)
├─ 长期维护 (0.15)
└─ 招聘难度 (0.05)

单属性评分 (0-10):

| 维度 | React | Vue | Angular |
|------|-------|-----|---------|
| 开发效率 | 8 | 9 | 7 |
| 生态系统 | 10 | 8 | 8 |
| 性能 | 8 | 8 | 7 |
| 学习成本 | 7 | 9 | 6 |
| 长期维护 | 9 | 8 | 8 |
| 招聘难度 | 9 | 7 | 6 |

效用计算:
React: 8.45, Vue: 8.30, Angular: 7.15

结论: React最优，Vue接近，可考虑团队熟悉度调整
```

### 场景2: 投资项目筛选

```
决策: 从10个项目中选择3个投资

维度:
├─ 财务回报 (NPV, IRR, 回收期) - 40%
├─ 风险水平 (市场, 技术, 团队) - 30%
├─ 战略价值 (协同, 定位, 学习) - 20%
└─ 实施可行性 (资源, 时间, 能力) - 10%

效用计算 + 敏感性分析:
- 基础权重: 项目A > B > C
- 权重±20%: 排序稳定
- 极端情景: 项目A仍最优

最终选择: A, B, C (效用排名前3)
```

## 常见陷阱

| 陷阱 | 表现 | 解决 |
|------|------|------|
| 维度遗漏 | 重要目标未考虑 | 利益相关者访谈 |
| 权重偏差 | 某指标权重过高 | 两两比较校准 |
| 数据质量差 | 评分凭感觉 | 量化指标优先 |
| 忽视相关性 | 指标间高度相关 | 降维或合并 |
| 静态分析 | 未考虑时间变化 | 情景分析 |
