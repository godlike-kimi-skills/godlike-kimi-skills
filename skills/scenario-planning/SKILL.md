# Scenario Planning 情景规划

**未来战略推演工具** - 基于 Shell Global Business Network 方法论

用于长期战略制定、黑天鹅防范、金融预警系统、不确定性管理。

---

## 核心框架

### 情景类型定义

```
基准情景 (Base Case)
└── 最可能发生的未来（概率 40-50%）

乐观情景 (Optimistic)
└── 有利因素叠加（概率 20-25%）

悲观情景 (Pessimistic)
└── 风险因素集中爆发（概率 20-25%）

极端风险 (Extreme/Black Swan)
└── 尾部风险事件（概率 5-10%）
```

### 情景构建流程

```
步骤 1: 识别关键驱动因素
    ├── PESTEL 分析
    ├── 行业特定因素
    └── 技术/社会趋势

步骤 2: 识别关键不确定性
    ├── 高影响 + 高不确定 = 情景轴
    └── 通常选择 2 个正交维度

步骤 3: 构建情景矩阵
    ├── 2×2 矩阵生成 4 个情景
    └── 为每个情景命名/叙事

步骤 4: 开发情景故事
    ├── 时间线推演
    ├── 关键事件序列
    └── 各情景成功/失败因素

步骤 5: 制定战略响应
    ├── 稳健策略（所有情景有效）
    ├── 情景特定策略
    └── 早期预警信号
```

## 情景矩阵模板

```
              不确定性维度 A
              低 ←────────────────→ 高
         ┌────────────┬────────────┐
    高   │   情景 1   │   情景 2   │
    ↑    │  "逆风前行" │  "黄金时代" │
维    ───┼────────────┼────────────┤
度    低   │   情景 3   │   情景 4   │
B   ↓    │  "稳定演进" │  "暗流涌动" │
         └────────────┴────────────┘
```

## 情景要素清单

每个完整情景应包含：

| 要素 | 内容 |
|------|------|
| **驱动因素** | 核心假设与驱动力 |
| **时间线** | 关键事件序列（1-3-5 年） |
| **关键指标** | 可量化的情景指标 |
| **预警信号** | 情景正在发生的早期指标 |
| **战略含义** | 对该情景的最优策略 |
| **触发行动** | 确认情景后的具体行动 |

## 应用场景

### 金融预警系统

```
情景构建示例：

维度 A: 通胀水平（低 / 高）
维度 B: 政策响应力度（温和 / 激进）

情景 1 - "滞胀困局"（高通胀+温和政策）
- 驱动：供应链持续紧张、能源价格上涨
- 指标：CPI>5%、GDP<2%、失业率上升
- 预警：核心PCE连续3月超预期
- 策略：配置实物资产、减少长债

情景 2 - "软着陆"（高通胀+激进政策）
- 驱动：联储快速加息成功抑制通胀
- 指标：CPI回落至3%、经济温和衰退
- 预警：就业数据放缓但有序
- 策略：质量因子、防御性板块

情景 3 - "金发女孩"（低通胀+温和政策）
- 驱动：供应链恢复、生产率提升
- 指标：CPI<2%、GDP>3%、充分就业
- 预警：制造业PMI持续改善
- 策略：风险资产、成长股

情景 4 - "通缩陷阱"（低通胀+激进政策）
- 驱动：政策过紧引发债务通缩
- 指标：CPI<0%、GDP负增长
- 预警：信贷增速骤降、企业破产增加
- 策略：长久期国债、现金为王
```
