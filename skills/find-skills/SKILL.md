# Find Skills

**技能发现与管理** - 借鉴 npm, pip, Homebrew 包管理器

技能搜索、版本管理、依赖解析、自动安装。

---

## 核心特性

### 📦 包管理功能

| 功能 | 说明 |
|------|------|
| **搜索** | 关键词、标签、分类搜索 |
| **安装** | 自动下载、解压、配置 |
| **更新** | 版本检查、增量更新 |
| **卸载** | 清理文件、移除配置 |
| **依赖** | 依赖树解析、冲突检测 |

### 🔍 发现机制

```
搜索维度:
├── 名称匹配 (模糊搜索)
├── 标签分类 (crypto, ai, security)
├── 评分排序 (下载量、评分)
├── 依赖过滤 (兼容性检查)
└── 版本筛选 (稳定版、最新版)
```

## 使用方法

### 搜索技能
```bash
# 关键词搜索
skills search crypto

# 分类浏览
skills list --category security

# 查看详情
skills info crypto-wallet
```

### 安装管理
```bash
# 安装技能
skills install crypto-wallet

# 更新技能
skills update crypto-wallet

# 卸载技能
skills remove crypto-wallet

# 列出已安装
skills list --installed
```

## 学习科学深度分析

### 🧠 间隔重复 (Spaced Repetition) 视角

Find Skills Skill 虽然是一个工具类技能，但其设计理念深刻体现了间隔重复的学习原理。技能发现过程本身就是一个"检索练习"——用户需要主动搜索、评估、选择合适的技能，这种主动检索比被动接受推荐更能形成持久记忆。

**检索式发现的优势**：
当用户使用`skills search crypto`时，他们不仅找到了需要的技能，还复习了搜索语法、分类体系、版本概念。每次搜索都是一次微型的间隔重复练习。研究表明，检索练习（retrieval practice）比单纯阅读能产生更强的记忆痕迹。

**渐进式暴露策略**：
技能的发现机制设计——从模糊搜索到分类浏览再到详情查看——对应间隔重复中的"渐进提示"技术。当用户记不清确切技能名称时，可以通过分类逐步缩小范围，这种分步提示降低了认知负荷，同时保持了主动学习的参与度。

**改进建议**：引入"技能推荐复习"机制，定期向用户推荐之前查看但未安装的技能（"您上周浏览了crypto-wallet，现在安装了吗？"），这利用了间隔重复中的"间歇提醒"原理。开发"技能使用历史"功能，展示用户已掌握技能的"熟练度曲线"，提示哪些技能需要"复习"（重新使用以巩固）。

### 🎯 刻意练习 (Deliberate Practice) 视角

Find Skills 为技能学习提供了完美的刻意练习基础设施——它让用户能够系统性地探索、安装、测试、评估技能，形成完整的能力提升闭环。

**技能学习的刻意练习循环**：
```
发现技能 → 安装试用 → 实际应用 → 评估反馈 → 掌握/卸载
    ↑                                         ↓
    └──────────── 持续改进 ←──────────────────┘
```

这个循环与刻意练习的核心要素一一对应：
- **明确目标**：每个技能都有明确的功能描述和使用场景
- **专注练习**：通过实际任务应用技能，而非简单阅读文档
- **即时反馈**：技能的成败立竿见影，用户立即知道是否有效
- **逐步进阶**：从简单技能开始，逐步尝试复杂的多依赖技能

**学习区管理**：
技能的标签分类（beginner, intermediate, advanced）帮助用户识别适合自己当前水平的学习内容，确保持续处于"学习区"——足够挑战以促进成长，又不至于因过难而挫败。

**改进建议**：建立"技能学习路径"功能，为不同角色（开发者、分析师、设计师）推荐从入门到精通的技能序列。引入"微挑战"机制——为每个技能设计3-5个渐进式练习任务，帮助用户从安装快速过渡到熟练应用。开发"技能组合训练"，推荐可以协同使用的技能组合（如crypto-wallet + dex-trader + price-monitor），通过多技能协同提升复杂问题解决能力。

### 🔄 迁移学习 (Transfer Learning) 视角

Find Skills 是学习迁移的放大器——它让用户能够快速将他人的经验和解决方案迁移到自己的场景中。

**知识迁移的三种形式**：
1. **近迁移**：同一领域内的技能迁移（如从crypto-wallet迁移到dex-trader）
2. **远迁移**：跨领域的模式迁移（如将从security技能学到的权限管理应用于finance技能）
3. **元迁移**：学习方法本身的迁移（掌握了如何安装使用crypto-wallet，就能更快学习任何新技能）

**依赖解析的迁移智慧**：
技能的依赖树解析机制本质上是在自动化迁移学习——当一个技能依赖其他技能时，系统自动识别并安装前置知识。这类似于学习高级数学时自动复习前置的代数知识。

**改进建议**：引入"技能相似度图谱"，可视化展示技能间的关联和可迁移性。当用户安装新技能时，自动提示"您已掌握的X技能与这个新技能有70%概念重叠，预计学习成本降低"。开发"跨技能模式库"，提取可复用的设计模式（如"轮询监控模式"在price-monitor、alert-manager、market-research中的应用），帮助用户识别和迁移通用解决方案。

### 🎓 元认知 (Metacognition) 视角

Find Skills 是元认知能力的外化工具——它帮助用户思考"我需要什么技能"、"我当前的能力缺口在哪里"、"如何最有效地学习新技能"。

**技能清单的自我认知功能**：
`skills list --installed` 不仅展示了已安装的技能，还隐含地映射了用户的能力画像。通过定期查看这个列表，用户可以反思：我擅长哪些领域？哪些领域存在盲区？这与元认知中的"自我评估"能力对应。

**搜索历史的元认知价值**：
用户的搜索历史是一面镜子，反映了他们的学习轨迹和兴趣演变。分析搜索模式可以发现：哪些概念反复出现（暗示理解困难）？哪些领域的技能增长最快？

**改进建议**：开发"技能健康仪表盘"，综合展示用户的能力分布、学习进度、知识缺口。引入"学习建议引擎"，基于用户的任务历史和已安装技能，主动推荐可能需要的技能（"检测到您近期频繁处理CSV文件，推荐安装csv-processor技能"）。建立"技能掌握度自评"功能，让用户定期评估自己对各技能的熟练程度，系统据此调整推荐策略。

### 🌱 成长思维 (Growth Mindset) 视角

Find Skills 的设计完全基于成长思维——它假设用户的能力是可以通过学习和实践不断发展的，而非固定不变的。

**能力可塑性的设计体现**：
- **无锁定设计**：用户可以随时安装、更新、卸载技能，没有"一旦选择就无法改变"的限制
- **版本演进**：技能的版本更新机制传达了"持续改进"的理念——今天的最佳实践明天可能会过时
- **依赖升级**：当依赖技能有更新时，系统提示升级，体现了"关联知识也需要同步成长"

**从失败中学习**：
安装失败、依赖冲突、版本不兼容——这些"错误"被设计为学习机会而非终点。错误消息不仅指出问题，还提供解决方案，帮助用户理解背后的原理。

**改进建议**：在技能描述中加入"学习曲线预估"和"常见初学者误区"，帮助用户建立合理预期。开发"技能成长故事"功能，展示其他用户从初学者到专家的学习历程。引入"技能挑战徽章"系统，奖励那些敢于尝试困难技能、从失败中学习的用户。建立"技能贡献指南"，鼓励用户将自己的技能分享给社区——教授他人是最高级的学习形式，也是成长思维的终极体现。

## 最佳实践

### 高效技能学习法

1. **主动检索**：使用搜索而非浏览来发现技能
2. **小步快跑**：先安装简单技能，建立信心后再尝试复杂技能
3. **立即应用**：安装后立即在实际任务中试用
4. **定期复习**：每周回顾已安装技能，巩固使用模式
5. **教以致用**：尝试向他人解释技能用法，加深理解
6. **迁移思考**：思考新技能与已掌握技能的关联

### 元认知自检清单

- [ ] 我清楚自己为什么需要这个技能吗？
- [ ] 这个技能与我已掌握的知识有关联吗？
- [ ] 我有明确的练习计划吗？
- [ ] 我如何评估自己是否掌握了这个技能？
- [ ] 如果遇到困难，我有备选方案吗？
