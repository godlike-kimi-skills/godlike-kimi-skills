# PDCA Cycle

**创始人**: Walter Shewhart → W. Edwards Deming  
**别名**: 戴明环、休哈特环  
**类型**: 执行与迭代  
**优先级**: P0

---

## 核心概念

### PDCA循环

```
        Plan (计划)
            ↓
    ┌───────────────┐
    │  明确目标     │
    │  分析现状     │
    │  找出原因     │
    │  制定措施     │
    │  制定计划     │
    └───────────────┘
            ↓
        Do (执行)
            ↓
    ┌───────────────┐
    │  实施计划     │
    │  执行措施     │
    │  收集数据     │
    │  记录过程     │
    └───────────────┘
            ↓
        Check (检查)
            ↓
    ┌───────────────┐
    │  对比结果     │
    │  评估效果     │
    │  找出偏差     │
    │  分析原因     │
    └───────────────┘
            ↓
        Act (处理)
            ↓
    ┌───────────────┐
    │  标准化       │
    │  推广成果     │
    │  遗留问题     │
    │  下一循环     │
    └───────────────┘
            ↓
        (回到Plan)

循环特点:
├─ 周而复始: 不断解决问题，持续改进
├─ 大环套小环: 整个组织一个大循环，各部门小循环
└─ 阶梯上升: 每循环一次，质量水平提升一步
```

## 详细实施指南

### Plan (计划) - 20%时间

```
步骤1: 明确问题
├─ 现状描述: 当前状态是什么?
├─ 期望状态: 应该是什么?
├─ 差距量化: 差距有多大?
└─ 重要性: 为什么需要解决?

步骤2: 现状分析
├─ 数据收集: 收集相关数据
├─ 流程梳理: 画出当前流程
├─ 问题定位: 问题出现在哪里?
└─ 根因分析: 使用5Why、鱼骨图

步骤3: 目标设定
├─ SMART原则:
│   ├─ S: Specific (具体的)
│   ├─ M: Measurable (可测量的)
│   ├─ A: Achievable (可实现的)
│   ├─ R: Relevant (相关的)
│   └─ T: Time-bound (有时限的)
└─ 例: 3个月内将客户投诉率从5%降至2%

步骤4: 制定对策
├─ 头脑风暴: 可能的解决方案
├─ 筛选评估: 可行性、成本、效果
├─ 制定措施: 具体行动计划
├─ 资源分配: 人、财、物、时间
└─ 风险预案: 可能的问题及应对

输出: 行动计划书
```

### Do (执行) - 10%时间

```
步骤1: 试点实施
├─ 小范围试点: 降低风险
├─ 按计划执行: 严格按照计划
├─ 实时监控: 及时发现问题
└─ 记录偏差: 计划vs实际

步骤2: 数据收集
├─ 过程数据: 执行过程中的数据
├─ 结果数据: 最终产出的数据
├─ 异常记录: 出现的问题
└─ 经验记录: 有价值的发现

关键原则:
├─ 按计划执行，不随意变更
├─ 完整记录，为Check做准备
└─ 遇到问题先记录，不急于解决
```

### Check (检查) - 15%时间

```
步骤1: 结果对比
├─ 目标 vs 实际: 达成率多少?
├─ 计划 vs 执行: 执行偏差?
├─ 定量分析: 数据对比
└─ 定性分析: 质的变化

步骤2: 效果评估
├─ 直接效果: 问题是否解决?
├─ 副作用: 产生了什么新问题?
├─ 成本效益: 投入产出比?
└─ 可持续性: 效果能否持续?

步骤3: 原因分析
├─ 成功经验: 为什么成功?
├─ 失败教训: 为什么失败?
├─ 偏差原因: 计划与执行差异?
└─ 外部因素: 环境变化影响?

输出: 检查报告
```

### Act (处理) - 5%时间

```
步骤1: 标准化
├─ 有效措施: 固化到流程/制度
├─ 文档更新: SOP、指南更新
├─ 培训推广: 相关人员培训
└─ 持续监控: 建立监控机制

步骤2: 遗留问题
├─ 未解决问题: 转入下一循环
├─ 新问题: 作为新的PDCA
└─ 优化空间: 持续改进点

步骤3: 总结分享
├─ 经验萃取: 可复用的方法
├─ 教训总结: 避免重复犯错
├─ 知识沉淀: 进入知识库
└─ 横向推广: 类似场景应用
```

## Kbot PDCA应用场景

### 场景1: 产品迭代

```
Plan:
├─ 问题: 用户留存率低于竞品
├─ 分析:  onboarding流程复杂
├─ 目标: 7日留存从40%→50%
└─ 对策: 简化注册流程，增加引导

Do:
├─ 设计新流程
├─ 开发A/B测试版本
├─ 小流量测试 (5%用户)
└─ 收集数据2周

Check:
├─ 结果: 留存率提升至48%
├─ 分析: 接近目标但未完全达成
├─ 发现: 用户在某步骤仍流失
└─ 原因: 该步骤提示不清

Act:
├─ 标准化: 新流程全量发布
├─ 遗留: 优化提示问题→下一循环
└─ 总结: 用户引导的关键发现
```

### 场景2: 运营优化

```
Plan:
├─ 问题: 获客成本CAC过高 ($100)
├─ 分析: 渠道效率低，转化差
├─ 目标: CAC降至$60
└─ 对策: 优化渠道组合，提升转化率

Do:
├─ 调整预算分配
├─ 优化落地页
├─ 测试新渠道
└─ 运行1个月

Check:
├─ 结果: CAC降至$65
├─ 分析: 有改善但未达目标
├─ 发现: 新渠道效果好但规模小
└─ 原因: 老渠道惯性大，调整不够

Act:
├─ 标准化: 新渠道增加预算
├─ 遗留: 老渠道优化→下一循环
└─ 调整目标: 下月CAC目标$55
```

### 场景3: 个人效率提升

```
Plan:
├─ 问题: 每天被琐事打断，重要工作拖延
├─ 分析: 缺乏时间块规划，干扰多
├─ 目标: 每天2小时深度工作时间
└─ 对策: 番茄工作法+时段隔离

Do:
├─ 上午9-11点设为深度时间
├─ 关闭通知，挂免打扰牌
├─ 番茄钟工作
└─ 执行2周

Check:
├─ 结果: 平均1.5小时/天深度工作
├─ 分析: 有改善但不够稳定
├─ 发现: 周三周五干扰最多
└─ 原因: 这两天会议多

Act:
├─ 标准化: 深度工作法继续
├─ 调整: 周三周五改到早7-9点
└─ 遗留: 会议优化→下一循环
```

## PDCA进阶技巧

### 1. 循环嵌套

```
大循环: 年度目标
    ├─ Q1循环
    │   ├─ 月循环
    │   │   ├─ 周循环
    │   │   │   └─ 日循环
    │   │   └─ ...
    │   └─ ...
    ├─ Q2循环
    └─ ...

原则: 大环指导小环，小环支撑大环
```

### 2. 循环加速

```
缩短循环周期:
├─ 传统: 年度→季度→月度
├─ 敏捷: 月度→双周→周
└─ 极限: 周→日→半日

适用: 快速变化环境，需要快速迭代
```

### 3. 8D问题解决法 (PDCA扩展)

```
D1: 组建团队
D2: 描述问题
D3: 实施临时措施
D4: 根本原因分析
D5: 选择永久措施
D6: 实施永久措施
D7: 预防再发
D8: 团队庆祝

本质: 结构化的PDCA，适用于复杂问题
```

## 常见错误

```
错误1: 只有P-D，没有C-A
表现: 做了很多事，从不总结
后果: 重复犯错，无法积累

错误2: C-A流于形式
表现: 检查走过场，处理敷衍
后果: 问题反复出现

错误3: 目标不具体
表现: "提升质量"、"做得更好"
后果: 无法评估效果

错误4: 循环周期过长
表现: 年度PDCA，反应太慢
后果: 错过改进时机

错误5: 不尊重数据
表现: 凭感觉判断效果
后果: 错误归因
```
