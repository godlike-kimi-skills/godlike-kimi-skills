### 2. 认识论堆栈 (Epistemic Stack)

```
层级架构分析:
┌─────────────────────────────────────────────────────────┐
│ Level 5: Actions (行动层)                                │
│   → 基于决策的具体行为                                    │
│   → 受所有下层影响                                        │
├─────────────────────────────────────────────────────────┤
│ Level 4: Decisions (决策层)                              │
│   → 基于信念的选择                                        │
│   → 涉及价值判断和权衡                                    │
├─────────────────────────────────────────────────────────┤
│ Level 3: Beliefs (信念层) ★ 核心                        │
│   → 对现实的概率化表示                                    │
│   → 可被证据更新                                          │
├─────────────────────────────────────────────────────────┤
│ Level 2: Evidence (证据层)                               │
│   → 观察和数据                                            │
│   → 受测量质量影响                                        │
├─────────────────────────────────────────────────────────┤

│   → 所有认识活动的目标                                    │
└─────────────────────────────────────────────────────────┘

关键洞见: 每个层级的质量都会影响其上所有层级
```

## 🎯 应用场景

### 1. 项目评估应用 (详细分析)

```
案例: "项目将产生$1000/月收入"

Phase 1: MVP构建完成
┌────────────────────────────────────────────────────────┐
│ 先验: 15% (类似项目基准成功率)                          │
│                                                        │
│ 证据分析:                                              │
│   P(E|Success) = 80%  (成功项目大多建了MVP)            │
│   P(E|Failure) = 40%  (失败项目也有部分建了MVP)        │
│   Bayes Factor = 2    (80/40)                          │
│                                                        │
│ 后验: 15% × 2 = 26%                                    │
│ 结论: 证据支持，但不足以达到行动阈值                    │
└────────────────────────────────────────────────────────┘

Phase 2: 获得首个客户
┌────────────────────────────────────────────────────────┐
│ 新先验: 26% (基于Phase 1后验)                          │
│                                                        │
│ 证据分析:                                              │
│   P(E|Success) = 90%  (成功项目几乎都获得了早期客户)    │
│   P(E|Failure) = 10%  (失败项目很少获得客户)           │
│   Bayes Factor = 9    (90/10)                          │
│                                                        │
│ 后验: 26% × 9 = 70%                                    │
│ 结论: ✅ 超过Proceed阈值(70%)，建议继续投入             │
└────────────────────────────────────────────────────────┘

阈值系统:
  Kill:    P(Success) < 30%
  Pivot:   30% ≤ P(Success) < 70%
  Scale:   P(Success) ≥ 70%
```

### 2. 校准游戏 (Calibration Game)

**机制分析:**
- 制作100个90%置信度的预测
- 校准良好的预测者应有90个正确
- 追踪和调整机制

**实用价值:**
- 识别过度自信/不足自信
- 长期提升预测准确性
- 量化认识论健康度

### 3. 认识论风险 (Epistemic Hazards)

| 风险 | 警告信号 | 防御策略 | 检测难度 |
|------|----------|----------|----------|
| 动机性推理 | 希望它为真 | 预先承诺标准 | 中 |
| 确认偏误 | 寻找支持证据 | 主动证伪 | 低 |
| 锚定效应 | 第一个数字主导 | 多锚点对比 | 低 |
| 可得性偏误 | 生动例子权重过高 | 基础概率 | 中 |
| 沉没成本 | 过去投资影响决策 | 评估边际价值 | 中 |
