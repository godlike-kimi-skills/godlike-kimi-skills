## 🎯 规划框架分析

### 1. 核心框架：战略制定七步法

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 愿景定义 (Vision)         → 长期目标设定                 │
│ 2. 现状诊断 (Current State)  → SWOT分析                     │
│ 3. 差距分析 (Gap Analysis)   → 目标vs现状                   │
│ 4. 路径设计 (Strategic Path) → 战略举措与里程碑             │
│ 5. 资源配置 (Resource Allocation) → 人财物时间优先级        │
│ 6. 里程碑设定 (Milestones)   → KR、时间表、责任人           │
│ 7. 风险管理 (Risk Management) → 风险识别与应对预案          │
└─────────────────────────────────────────────────────────────┘
```

### 2. 融合方法论

| 方法论 | 应用环节 | 价值贡献 |
|--------|----------|----------|
| **Hoshin Kanri (方针管理)** | 目标分解与对齐 | 纵向贯通（愿景→部门→个人） |
| **OKR** | 目标设定与追踪 | 定量可衡量的关键结果 |
| **Catchball** | 沟通过程 | 双向反馈，确保可行性 |
| **SWOT** | 现状诊断 | 全面评估内外部环境 |

### 3. 框架优势

- ✅ **可落地性强**：输出具体的战略地图而非空泛分析
- ✅ **层级对齐**：从3-5年愿景到季度OKR的完整贯通
- ✅ **模板完整**：提供可直接使用的输出模板
- ✅ **风险控制**：包含风险管理环节

### 4. 框架局限

- ❌ 缺少**动态调整机制**（如何应对环境变化）
- ❌ 缺少**战略解码**的具体方法（如何将战略转化为行动）
- ❌ 缺少**竞争分析**的深入框架（波特五力等）
- ❌ 缺少**财务建模**相关内容

## 🧠 决策方法

### 1. 定性决策方法

| 方法 | 应用场景 | 决策逻辑 |
|------|----------|----------|
| **SWOT分析** | 现状诊断 | 内外部因素交叉分析 |
| **差距分析** | 战略路径设计 | 目标状态 - 当前状态 = 行动方向 |
| **Catchball协商** | 目标分解 | 上下级协商确定可行目标 |

### 2. 定量决策方法

| 方法 | 应用场景 | 决策逻辑 |
|------|----------|----------|
| **信心指数** | OKR制定 | 初始50%（有挑战但可实现） |
| **领先/滞后指标** | 监控体系 | 区分驱动指标与结果指标 |
| **风险概率×影响** | 风险评估 | 量化风险优先级 |

### 3. 决策流程

```
愿景定义 → 现状诊断 → 差距识别 → 路径设计 → 资源配置 → 里程碑设定 → 风险管控
    ↑______________________________________________________________↓
                        (持续监控与调整)
```

```
1. 使用【战略制定七步法】进行年度规划
```

### 应用场景2：新产品战略

```
流程:
1. 愿景定义：3年内成为细分市场Top3
2. 现状诊断：SWOT分析竞争格局

4. 路径设计：产品路线图
```

```
4. 战略地图可视化展示
```
