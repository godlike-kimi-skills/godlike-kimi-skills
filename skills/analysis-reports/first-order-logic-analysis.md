## 推理方法分析

### 核心推理方法

| 方法 | 描述 | 应用场景 |
|------|------|----------|
| **假言推理 (Modus Ponens)** | P→Q, P ⊢ Q | 从条件和事实推出结论 |
| **拒取式 (Modus Tollens)** | P→Q, ¬Q ⊢ ¬P | 否定结论推出否定前提 |
| **三段论 (Syllogism)** | ∀x(P(x)→Q(x)), P(a) ⊢ Q(a) | 从一般规律推导具体实例 |
| **消解原理 (Resolution)** | P∨Q, ¬P∨R ⊢ Q∨R | 自动定理证明 |

### 论证分析框架

```
┌─────────────────────────────────────────────┐
│ 1. 识别谓词和个体                            │
│ 2. 形式化表达 (符号化)                       │
│ 3. 检验形式有效性                            │
│ 4. 验证前提真实性                            │
│ 5. 识别隐含假设                              │
│ 6. 评估结论可信度                            │
└─────────────────────────────────────────────┘
```

### 形式谬误识别能力

- **肯定后件** (Affirming the Consequent)
- **否定前件** (Denying the Antecedent)
- **四词项错误** (Four-term Fallacy)
- **不当换位** (Illicit Conversion)

| **程序验证** | 形式化验证算法正确性 | 前置条件、后置条件、不变式分析 |

### 集成使用示例

```
复杂决策场景分析:

Step 1: First-Order Logic
  └─ 将业务论证形式化
  └─ 识别: 前提、结论、隐含假设

Step 2: Bad Arguments
  └─ 检查是否有逻辑谬误
  └─ 识别: 稻草人、假两难等

Step 3: Causal Inference
  └─ 验证因果关系的有效性
  └─ 排除: 混淆变量、选择偏差

Step 4: Razors Laws
  └─ 在多个合理解释中选择最简单的
  └─ 应用: 奥卡姆剃刀
```

## 总结

**First-Order Logic** Skill是决策系统中的**结构验证层**，负责：
- 确保论证的形式有效性
- 识别隐含假设和逻辑漏洞
- 为其他推理方法提供严格的逻辑基础

**核心价值**: 在AI决策系统中，它是防止"逻辑错误导致决策失败"的关键防线。

**推荐集成深度**: 高 - 建议作为所有复杂决策的前置验证步骤
