## 🔗 关键关联分析

### 强关联组

#### 1. Cybernetics ↔ Dissipative Structures
| 关联维度 | 说明 |
|----------|------|
| **理论基础** | 控制论的信息视角 + 耗散结构的熵视角 = 完整系统观 |
| **互补性** | 控制"如何"维持有序 + 演化"为何"产生有序 |
| **建议** | 建立显式链接，统一"负熵输入"与"控制机制"概念 |

#### 2. Loopy Systems ↔ System Dynamics
| 关联维度 | 说明 |
|----------|------|
| **内容重叠** | ~70%内容重复 (Stocks/Flows, Feedback Loops) |
| **定位差异** | Loopy轻量/可视化 vs SD学术/仿真 |
| **建议** | 明确区分或合并 |

### 弱关联组

#### Loopy Systems ↔ Dissipative Structures
- 理论视角差异大
- 一个重操作，一个重哲学
- 建议：保持独立，但可添加"如何使用Loopy可视化耗散结构"的链接

## 🎯 核心洞察

### 洞察1: 理论互补性
4个Skills覆盖系统思维的4个维度:
- **控制论**: 系统"如何"被控制
- **耗散结构**: 系统"为何"演化
- **系统动力学**: 系统"怎样"行为
- **Loopy**: 系统"如何"可视化

### 洞察2: 内容分层
```
哲学层: Dissipative Structures
       ↓
理论层: Cybernetics, System Dynamics
       ↓
工具层: Loopy Systems
       ↓
基础层: System Thinking
```

### 洞察3: 使用场景
| 场景 | 推荐Skill |
|------|-----------|
| 设计控制系统 | Cybernetics |

| 快速概念探索 | Loopy Systems |
