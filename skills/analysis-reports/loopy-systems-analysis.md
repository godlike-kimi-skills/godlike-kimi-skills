## 🏛️ 系统理论框架

### 核心理论：因果回路建模 (Causal Loop Modeling)

```
┌─────────────────────────────────────────────────────────────┐
│                  Loopy Systems 理论架构                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  基础概念                                                    │
│  ├─ Stocks (存量): 累积量，变化缓慢，有记忆                  │
│  │   └─ 例如: 用户基数、收入、库存、声誉                     │
│  └─ Flows (流量): 变化率，可快速改变                         │
│      └─ 例如: 新用户/天、流失率、支出率                      │
│                                                             │
│  反馈回路类型                                                │
│  ├─ Reinforcing Loops (R): 强化循环，放大变化                │
│  │   └─ 病毒式增长、复利、军备竞赛                           │
│  └─ Balancing Loops (B): 平衡循环，趋于稳定                  │
│      └─ 恒温器调节、市场饱和、资源限制                       │
│                                                             │
│  系统原型 (Archetypes)                                       │
│  ├─ Limits to Growth (增长极限)                             │
│  ├─ Shifting the Burden (转嫁负担)                          │
│  ├─ Tragedy of the Commons (公地悲剧)                       │
│  └─ Success to the Successful (富者愈富)                    │
│                                                             │
│  杠杆点 (Leverage Points)                                    │
│  └─ 从范式到参数，9个干预层次                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 理论特色

1. **可视化导向**: 强调因果回路图的绘制
2. **易学易用**: 比系统动力学更轻量、更直观
3. **商业聚焦**: 紧密围绕业务场景设计
4. **系统原型**: 提供常见模式的快速识别
