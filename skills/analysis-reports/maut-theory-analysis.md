## 🎯 规划框架分析

### 1. 核心理论框架

```
多属性效用函数:
U(X) = U(X₁, X₂, ..., Xₙ)

可加性效用函数 (最常见):
U(X) = Σ wᵢ × uᵢ(Xᵢ)

其中:
- X = 方案
- Xᵢ = 第i个属性的表现
- wᵢ = 第i个属性的权重 (Σwᵢ = 1)
- uᵢ = 第i个属性的效用函数 (0-1标准化)
```

### 2. 理论贡献

| 问题 | MAUT解决方案 | 理论创新 |
|------|--------------|----------|
| 多目标不可比 | 转化为统一效用值 | 效用理论量化 |
| 主观判断不一致 | 结构化权重分配 | Swing权重法 |
| 群体决策难共识 | 客观计算排序 | 数学化决策支持 |
| 风险偏好差异 | 非线性效用函数 | 指数/对数函数 |

### 3. MAUT实施五步骤

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤1: 目标层次结构化                                        │
│        ├─ 一级维度: 财务收益 (w=0.4)                        │
│        │  ├─ 指标1: NPV (w₁=0.6)                            │
│        │  ├─ 指标2: IRR (w₂=0.3)                            │
│        │  └─ 指标3: 回收期 (w₃=0.1)                         │
│        └─ ...其他维度                                        │
├─────────────────────────────────────────────────────────────┤
│ 步骤2: 确定权重                                              │
│        ├─ 直接分配法                                         │
│        ├─ 两两比较法 (AHP)                                   │
│        └─ Swing权重法 (推荐)                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤3: 构建单属性效用函数                                    │
│        ├─ 线性函数 (风险中性)                                │
│        ├─ 指数函数 (风险厌恶)                                │
│        └─ 对数函数 (强风险厌恶)                              │
├─────────────────────────────────────────────────────────────┤
│ 步骤4: 计算总效用                                            │
│        U(X) = Σ wᵢ × uᵢ(Xᵢ)                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤5: 一致性检验                                            │
│        ├─ 权重和检验                                         │
│        ├─ 敏感性分析                                         │
│        └─ 极端值检验                                         │
└─────────────────────────────────────────────────────────────┘
```

### 4. 框架优势

- ✅ **理论严谨**：基于Keeney & Raiffa经典决策理论
- ✅ **数学基础扎实**：效用函数、权重计算有明确公式
- ✅ **风险考虑**：支持风险态度建模（风险厌恶/中性/追求）
- ✅ **完整性高**：从理论到实施到陷阱全覆盖
- ✅ **实用性强**：提供两个详细应用场景

### 5. 框架局限

- ❌ **计算复杂度**：大规模决策时计算繁琐
- ❌ **主观性仍存**：权重确定依赖专家判断
- ❌ **独立性假设**：假设属性间相互独立（现实中常不满足）
- ❌ **数据要求高**：需要大量量化数据

## 🧠 决策方法

### 1. 权重确定方法对比

| 方法 | 复杂度 | 客观性 | 适用场景 |
|------|--------|--------|----------|
| **直接分配法** | 低 | 低 | 简单决策、专家经验丰富 |
| **两两比较法** | 中 | 中 | 需一致性检验的决策 |
| **Swing权重法** | 高 | 高 | 重要战略决策（推荐） |

### 2. 效用函数类型选择

| 类型 | 公式 | 适用风险偏好 | 典型应用 |
|------|------|--------------|----------|
| **线性** | u(x)=(x-x_min)/(x_max-x_min) | 风险中性 | 成本、效率指标 |
| **指数** | u(x)=(1-e^(-rx))/(1-e^(-r)) | 风险厌恶 | 投资回报 |
| **对数** | u(x)=ln(x/x_min)/ln(x_max/x_min) | 强风险厌恶 | 高风险项目 |

### 3. 一致性检验方法

```
检验项目:

1. 权重和检验: Σwᵢ = 1.0 ✓

2. 敏感性分析:
   - 权重±20%变化时，排序是否稳定?
   - 若不稳定，需重新审视权重

3. 极端值检验:
   - 检查各方案在各指标上的表现
   - 识别是否有"一票否决"项

4. 逻辑一致性:
   - 直觉判断与计算结果是否一致?
   - 不一致时检查模型假设
```

### 4. 决策流程

```
定义决策问题 → 识别评估维度 → 确定权重 → 构建效用函数 → 计算效用 → 一致性检验 → 方案排序
      ↑                                                                            ↓
      └────────────────── (敏感性分析与调整) ← 结果解释与沟通 ← 决策建议 ←─────────┘
```

## 💡 实际项目应用方式

### 应用场景1：技术方案选型

```
决策: 选择前端框架

候选: React vs Vue vs Angular

维度与权重:
├─ 开发效率 (0.25)
├─ 生态系统 (0.20)
├─ 性能表现 (0.20)
├─ 学习成本 (0.15)
├─ 长期维护 (0.15)
└─ 招聘难度 (0.05)

单属性评分 (0-10):

| 维度 | React | Vue | Angular |
|------|-------|-----|---------|
| 开发效率 | 8 | 9 | 7 |
| 生态系统 | 10 | 8 | 8 |
| 性能 | 8 | 8 | 7 |
| 学习成本 | 7 | 9 | 6 |
| 长期维护 | 9 | 8 | 8 |
| 招聘难度 | 9 | 7 | 6 |

效用计算:
React: 8.45, Vue: 8.30, Angular: 7.15

结论: React最优，Vue接近，可考虑团队熟悉度调整
```

### 应用场景2：投资项目筛选

```
决策: 从10个项目中选择3个投资

维度:
├─ 财务回报 (NPV, IRR, 回收期) - 40%
├─ 风险水平 (市场, 技术, 团队) - 30%
├─ 战略价值 (协同, 定位, 学习) - 20%
└─ 实施可行性 (资源, 时间, 能力) - 10%

效用计算 + 敏感性分析:
- 基础权重: 项目A > B > C
- 权重±20%: 排序稳定
- 极端情景: 项目A仍最优

最终选择: A, B, C (效用排名前3)
```

### 应用场景3：供应商评估

```
流程:
1. 识别评估维度：价格、质量、交付、服务、战略匹配
2. 使用Swing法确定权重
3. 构建各指标的效用函数
4. 收集各供应商在各维度的表现数据
5. 计算各供应商总效用
6. 进行敏感性分析
7. 生成决策建议
```
