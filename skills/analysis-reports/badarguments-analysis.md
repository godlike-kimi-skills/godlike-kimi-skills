## 推理方法分析

### 24种逻辑谬误分类体系

| 类别 | 谬误数量 | 核心问题 | 典型示例 |
|------|----------|----------|----------|
| **相关性谬误** | 5种 | 论证与结论不相关 | 人身攻击、诉诸权威 |
| **预设谬误** | 4种 | 前提存在问题 | 假两难、循环论证 |
| **因果谬误** | 3种 | 因果关系错误 | 后此谬误、相关即因果 |
| **歧义谬误** | 2种 | 语言含糊不清 | 歧义、语法歧义 |
| **统计谬误** | 4种 | 统计推理错误 | 基础比率忽视、赌徒谬误 |
| **认知谬误** | 6种 | 认知偏差导致 | 确认偏误、沉没成本 |

### 谬误检测协议

```
When evaluating an argument:

1. IDENTIFY the conclusion      (识别结论)
2. LIST the premises            (列出前提)
3. CHECK each fallacy type      (检查每种谬误)
4. ASSESS the severity          (评估严重程度)
5. DECIDE: Valid, Flawed, or Reject  (决策)
```

### 核心谬误详解

#### 高频率谬误 (Top 6)

| 谬误 | 定义 | 决策场景 | 防护策略 |
|------|------|----------|----------|
| **确认偏误** | 寻找证实，忽视证伪 | 投资决策、方案选择 | 主动证伪、红队思维 |
| **沉没成本** | 因已投入而继续 | 项目终止决策 | 只看未来价值 |
| **可得性启发** | 以易得例子判断 | 风险评估 | 寻找系统数据 |
| **幸存者偏差** | 只看成功案例 | 策略学习 | 研究失败同等 |
| **仓促概括** | 小样本推一般 | 策略验证 | 统计显著性 |
| **虚假两难** | 非此即彼 | 方案选择 | 寻找第三选择 |

#### 商业决策中的高危谬误

```
Pre-Launch Check:
├─ Hasty generalization? [Check sample size]
├─ Survivorship bias? [Check failure rate]
├─ Confirmation bias? [Seek disconfirming evidence]
└─ False dichotomy? [Look for third ways]

Decision Review:
├─ Sunk cost fallacy? [Is past clouding judgment?]
├─ Availability heuristic? [Is recent failure overweighted?]
└─ Anchoring? [Is original plan too sticky?]
```

### 集成工作流

```
决策论证质量检查流程:

输入: 决策建议方案

Step 1: First-Order Logic
  └─ 检查论证的形式有效性
  └─ 识别: 肯定后件、否定前件等形式谬误

Step 2: Bad Arguments
  └─ 检查24种非形式谬误
  └─ 重点: 确认偏误、沉没成本、幸存者偏差

Step 3: Causal Inference
  └─ 检查因果主张的有效性
  └─ 排除: 后此谬误、相关即因果

Step 4: Razors Laws
  └─ 简化后的论证是否仍然成立?
  └─ 检查是否有过度复杂化的谬误

Step 5: 决策
  └─ 基于清理后的论证做出决策
  └─ 记录检测到的谬误作为学习

输出: 谬误检测报告 + 清理后的决策建议
```

### 改进建议

| 优先级 | 建议 | 预期效果 |
|--------|------|----------|
| **高** | 添加Kbot检测命令和输出格式 | 提升与First-Order Logic的一致性 |
| **高** | 补充谬误严重性评估标准 | 区分致命谬误和轻微瑕疵 |
| **中** | 增加更多中国/亚洲商业案例 | 增强本地化适用性 |
| **中** | 补充谬误组合检测 | 识别多谬误叠加的复杂情况 |
| **中** | 添加谬误修复建议 | 不仅识别，还要纠正 |
| **低** | 补充更多认知科学背景 | 解释谬误的心理机制 |

### 与First-Order Logic的差异与互补

| 维度 | First-Order Logic | Bad Arguments |
|------|-------------------|---------------|
| **关注焦点** | 论证结构 | 论证内容 |
| **谬误类型** | 形式谬误 | 非形式谬误 |
| **分析方法** | 符号化、形式化 | 语义分析、案例对照 |
| **适用场景** | 结构化论证 | 日常推理、讨论 |
| **学习曲线** | 较陡 | 较平缓 |

### 潜在局限

| 局限 | 描述 | 缓解策略 |
|------|------|----------|
| **谬误误用** | 错误地将有效论证标记为谬误 | 与First-Order Logic交叉验证 |
| **过度警惕** | 对所有论证都怀疑 | 设置谬误严重性阈值 |
| **遗漏新型谬误** | 新出现的推理错误 | 定期更新谬误库 |
| **文化差异** | 某些谬误在不同文化中表现不同 | 添加文化敏感性说明 |

## 总结

**Bad Arguments** Skill是决策系统中的**谬误检测层**，负责：
- 识别24种常见逻辑谬误
- 防止认知偏差影响决策
- 提升论证质量和讨论效率

**核心价值**: 在AI决策系统中，它是防止"非理性推理导致错误决策"的关键防线。

**推荐集成深度**: 高 - 建议在所有涉及论证评估的场景使用

**关键洞察**: 
> "识别谬误不是为了赢得争论，而是为了接近真理。"

**与First-Order Logic的关系**:
- First-Order Logic是"语法检查器"，确保论证结构正确
- Bad Arguments是"内容审核员"，确保论证内容可靠
- 两者结合，实现对论证的全面质量控制

**适用性评级**:
- 日常决策: ⭐⭐⭐⭐⭐
- 团队讨论: ⭐⭐⭐⭐⭐
- 复杂分析: ⭐⭐⭐⭐ (需配合其他Skills)
- 形式验证: ⭐⭐⭐ (First-Order Logic更适合)
