# Razors Laws (剃刀定律体系)

**核心**: 奥卡姆剃刀、汉隆剃刀、希钦斯剃刀  
**类型**: 逻辑与认知  
**优先级**: P1

---

## 剃刀定律体系

### 1. 奥卡姆剃刀 (Occam's Razor)

**原句**: "如无必要，勿增实体"
**含义**: 在多个解释中，最简单的通常是最好的

```
应用场景:

问题: 网站流量突然下降

复杂解释:
├─ 竞争对手发起了秘密攻击
├─ 搜索引擎算法有隐藏更新
├─ 用户行为发生根本性转变
└─ 需要聘请咨询公司深度分析

简单解释:
└─ 服务器昨天宕机了2小时

奥卡姆剃刀建议:
先验证简单解释，再考虑复杂解释
```

**数学表达**:
```
贝叶斯奥卡姆剃刀:

P(H|D) ∝ P(D|H) × P(H)

简单假设H₁的先验概率P(H₁) > 复杂假设H₂的P(H₂)
因为简单假设的"假设空间"更小

例: 数据拟合
├─ 简单模型: y = ax + b (2参数)
├─ 复杂模型: y = ax⁵ + bx⁴ + ... (6参数)
└─ 剃刀: 除非复杂模型显著更好，否则选简单
```

### 2. 汉隆剃刀 (Hanlon's Razor)

**原句**: "永远不要把可以愚蠢解释的事情归因于恶意"
**含义**:  cock-up theory > conspiracy theory

```
应用场景:

情况: 同事没有回复你的邮件

恶意解释:
├─ 他在故意忽视我
├─ 他想抢我的功劳
└─ 他在背后说我坏话

愚蠢解释:
├─ 他邮件太多淹没了
├─ 他忘记标记待办
└─ 他以为已经回复了

汉隆剃刀建议:
默认假设是愚蠢/疏忽，而非恶意
除非有强证据证明恶意
```

**商业应用**:
```
客户投诉处理:

产品出问题的解释:
├─ 恶意: 供应链故意 sabotage
├─ 愚蠢: 质检流程有漏洞

剃刀建议: 先检查流程，再考虑人为破坏

结果: 95%的问题是流程/疏忽，5%才是恶意
```

### 3. 希钦斯剃刀 (Hitchens' Razor)

**原句**: "可以被无证据断言的，可以被无证据否定"
**含义**: 举证责任在提出主张的一方

```
应用场景:

断言: "我们的竞争对手在窃取我们的客户数据"

举证责任:
├─ 谁提出谁举证
├─ 若无证据，该断言可被否定
└─ 不需要"证明没有窃取"

实践:
└─ 要求提供证据，否则不采信
```

**会议讨论**:
```
断言: "如果我们不降价，就会失去50%客户"

希钦斯剃刀:
├─ 举证: 客户调研数据?
├─ 举证: 历史案例分析?
├─ 举证: 客户流失预警指标?
└─ 若无证据，该主张不成立

替代: "我们需要验证客户对价格的敏感度"
```

### 4. 萨根标准 (Sagan Standard)

**原句**: "非凡的主张需要非凡的证据"
**含义**: 主张越离奇，证据门槛越高

```
非凡程度梯度:

低: "下个月销售额会增长5%"
   └─ 需要: 历史数据支持

中: "新产品能占领20%市场份额"
   └─ 需要: 市场调研、竞品分析

高: "这个策略将颠覆整个行业"
   └─ 需要: 技术突破证明、用户验证

极高: "AI将在3年内取代所有程序员"
   └─ 需要: 技术里程碑、实际替代案例
```

### 5. 格赖斯剃刀 (Grice's Razor)

**原句**: "选择更简洁的对话解释"
**含义**: 假设说话者是合作且理性的

```
沟通应用:

同事说: "这个方案可能有问题"

不合作解释:
├─ 他在质疑我的能力
├─ 他想推翻我的方案
└─ 他在向老板告密

合作解释:
└─ 他真的发现了潜在问题

剃刀: 假设合作意图，除非有证据反对
```

## 综合应用框架

### 问题分析流程

```
遇到问题:
    ↓
步骤1: 奥卡姆剃刀
    ├─ 最简单的解释是什么?
    └─ 先验证最简单的
    ↓
步骤2: 汉隆剃刀
    ├─ 是否可能是愚蠢/疏忽?
    └─ 默认非恶意，除非证据
    ↓
步骤3: 希钦斯剃刀
    ├─ 各主张的证据是什么?
    └─ 无证据的主张→否定
    ↓
步骤4: 萨根标准
    ├─ 主张的非凡程度?
    └─ 证据是否匹配?
    ↓
结论: 剃刀后的简洁解释
```

### 决策检查清单

```
决策前剃刀检查:

□ 奥卡姆: 是否选择了最简单的可行方案?
□ 汉隆: 是否过度归因于恶意?
□ 希钦斯: 是否有证据支持关键假设?
□ 萨根: 证据强度是否与主张匹配?
□ 格赖斯: 是否假设了合作意图?

通过所有检查 → 决策更可靠
```

## Kbot剃刀应用

### /razor 命令

```
/razor "问题或陈述"

输出:
┌─────────────────────────────────────────────┐
│ 剃刀分析                                    │
│ 问题: [原文]                                │
├─────────────────────────────────────────────┤
│ 奥卡姆剃刀                                  │
│ ├─ 简单解释: [最简单的解释]                 │
│ ├─ 复杂解释: [复杂的替代解释]               │
│ └─ 建议: [先验证简单解释]                   │
├─────────────────────────────────────────────┤
│ 汉隆剃刀                                    │
│ ├─ 愚蠢/疏忽解释: [非恶意解释]              │
│ ├─ 恶意解释: [阴谋论解释]                   │
│ └─ 建议: [默认非恶意]                       │
├─────────────────────────────────────────────┤
│ 希钦斯剃刀                                  │
│ ├─ 主张: [陈述中的断言]                     │
│ ├─ 证据: [现有证据]                         │
│ └─ 结论: [证据不足则否定]                   │
├─────────────────────────────────────────────┤
│ 萨根标准                                    │
│ ├─ 主张非凡程度: [低/中/高]                 │
│ ├─ 证据强度: [弱/中/强]                     │
│ └─ 匹配度: [匹配/不匹配]                    │
├─────────────────────────────────────────────┤
│ 剃刀后结论                                  │
│ [简洁、有证据支持的解释]                    │
└─────────────────────────────────────────────┘
```

### 应用示例

```
输入: "客户突然取消订单，肯定是竞品挖走了"

输出:
├─ 奥卡姆: 可能是内部审批延迟
├─ 汉隆: 默认非恶意竞争，可能是客户自身原因
├─ 希钦斯: "竞品挖走"的证据是? (无)
├─ 萨根: 阴谋论级别主张，需要强证据
└─ 结论: 先联系客户了解真实原因
```
